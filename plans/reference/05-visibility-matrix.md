# 05 — مصفوفة الرؤية

> هذه مصفوفة **منطق رؤية بيانات** إضافية فوق `abilities` العادية (انظر [04-permissions](04-permissions.md)) — تتحكم بما يُعرَض من حقول المنسق/المراقب داخل نفس صفحة المشروع، وليس بمن يملك صلاحية view أصلاً.

## المشكلة التي تحلّها

المشروع الواحد يمر بأطراف متعددة (مدير مشروع، منسق، مدير دائرة، مدير رقابة، مراقب). قرار تصميمي متعمَّد: **المراقب لا يرى بيانات أو هوية المنسق ولا ما عبّأه بقائمة التحقق**، وبالمثل **مدير المشروع ومدير الدائرة لا يريان بيانات المراقب**. هذا العزل يُطبَّق على مستوى الاستعلام/الـ Controller، وليس فقط بإخفاء عناصر بالـ Blade (لتفادي تسرّب البيانات عبر HTML/Response).

## الدوال المركزية في `Project`

```php
Project::scopeVisibleToUser(Builder $query, ?User $user)   // فلترة القوائم — أي مشاريع تظهر لهذا المستخدم أصلاً
Project::isVisibleToUser(?User $user): bool                  // نفس المنطق لسجل واحد
Project::showsCoordinatorDataTo(?User $user): bool            // هل يُعرَض عمود/بيانات المنسق؟
Project::showsMonitorDataTo(?User $user): bool                // هل يُعرَض عمود/بيانات المراقب؟
Project::isAssignedMonitor(?User $user): bool                 // هل هذا المستخدم هو المراقب المعيَّن تحديداً؟
```

`super_admin` يتجاوز كل شيء في الدوال الأربع الأولى (يرى كل شيء دائماً).

## من يرى ماذا (as-built، مستخرج من `app/Models/Project.php`)

| الدور | مشاريع مرئية (`scopeVisibleToUser`) | يرى بيانات المنسق؟ | يرى بيانات المراقب؟ |
|---|---|---|---|
| `project_manager` | مشاريعه فقط (`project_manager_id`) | ✅ فقط إن كان مشروعه | ❌ أبداً |
| `department_manager` | مشاريع مديري مشاريع تابعين لدائرته | ✅ عبر `approvableByDepartmentManager()` | ❌ أبداً |
| `coordinator` | المشاريع المُسنَد لها كمنسق (`coordinator_id`) | ✅ (هو نفسه) | لا ينطبق (لا يصل لمرحلة المراقب أصلاً) |
| `monitor` | المشاريع المُسنَد لها كمراقب (`monitor_person_id`) | ❌ أبداً | ✅ فقط إن كان المراقب المعيَّن |
| `monitoring_director` | كل المشاريع (`default => $query`) | ✅ دائماً | ✅ دائماً |
| `general_management` | كل المشاريع | ✅ دائماً | ✅ دائماً |
| بلا `person` مرتبط | لا شيء (`whereRaw('1=0')`) | ❌ | ❌ |

## تصحيح نطاق مدير الرقابة العامة (as-built)

ملاحظة تخطيط قديمة اعتبرت مدير الرقابة "خارج التفاعل مع بيانات المنسق". **التصحيح الفعلي:** مدير الرقابة العامة **يستطيع تعديل بيانات المشروع** (بما فيها الاطلاع الكامل)، لكن دون أن يكون هو من *يعبّئ* حقول المنسق ابتداءً (تلك مسؤولية المنسق/مدير المشروع نيابةً) — الفرق بين "رؤية/تعديل إداري" و"تعبئة أصلية لعمود طرف آخر".

## شاشة عمل المراقب (`monitor-work`)

`ProjectController::monitorWork()` + `monitor-work.blade.php` — Controller/View مخصصان منفصلان عن صفحة `show` العامة، **لا يُحمِّلان** `coordinator_id` أو `coordinator_value` إطلاقاً (وليس فقط إخفاءهما بالـ Blade). طُبِّق هذا الفصل على مستوى الكود منذ البداية تنفيذاً لقرار "عزل رؤية المراقب".

## بعد تعبئة المنسق: رؤية مدير المشروع

بعد أن يعبّئ المنسق عموده، **مدير المشروع يرى تعبئة المنسق عرضاً فقط** (لا تعديل) — يظهر ضمن `_project_summary` بعد اكتمال تلك المرحلة، دون منحه صلاحية تعديل عمود لا يخصه.
