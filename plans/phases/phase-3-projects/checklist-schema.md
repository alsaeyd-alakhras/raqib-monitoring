## منطق حساب نسب الجاهزية

> **مستخرج من معادلة Excel الفعلية** (ورقة "تقرير مشروع"، الخلية B34/B35):
> `LET(s, العمود, req, COUNT(s<>"غير مطلوب"), IF(req=0,"", SUMPRODUCT((s="جاهز") + (s="جزئي")*0.5) / req))`

### لكل مجموعة (coordinator أو monitor):

```
نسبة_مجموعة =
    (عدد "جاهز"  +  0.5 × عدد "جزئي")
    ÷
    (إجمالي_البنود − عدد "غير مطلوب")
    × 100
```

- **"جزئي" يُحتسب بوزن 0.5** (نصف بند جاهز) — هذا الخيار كان مفقوداً في النسخة الأولى وأُضيف بعد فحص معادلة Excel.
- **"غير مطلوب" يُستثنى من المقام** (لا ينقص النسبة ظلماً).
- إذا كان المقام = 0 (كل البنود "غير مطلوب")، النسبة فارغة/100% (المجموعة مكتملة بالاستثناء).

### حالة الجاهزية المشتقة (3 مستويات — من معادلة Excel A52، تعتمد عمود المراقب):
```
يوجد "غير جاهز"  → 🔴 موقوف — يحتاج مراجعة
يوجد "جزئي"      → 🔶 جاهز جزئياً — يحتاج متابعة
غير ذلك          → ✅ جاهز للتنفيذ — موصى بالمتابعة
```
تُخزَّن/تُعرض في حقل `حالة الجاهزية` (مرتبط بـ open-questions نقطة 6 — مشتقة من المشروع، لا تُدخَل يدوياً).

### النسبة الإجمالية:

```
نسبة_إجمالية = متوسط نِسَب كل المجموعات النشطة
```

> **قرار تصميمي:** المتوسط البسيط (غير المرجَّح) كافٍ حالياً — يتوافق مع ما ظهر في ملف Excel. إن أراد العميل أوزاناً مستقبلاً، أضف `weight` لجدول `checklist_groups`.

### نقل النسبة إلى KPI:

```
monitoring_activities.execution_value = projects.monitor_readiness_pct
```
يُحدَّث تلقائياً كلما عدّل المراقب قائمة التحقق (via Model Observer أو explicit update في Controller).

## البيانات الأولية للمجموعات والبنود (Seed Data)

> البنود التالية **منسوخة حرفياً من ورقة "تقرير مشروع" في `project.xlsx`** (وليست تخميناً).

### مجموعة 1: 📦 اللوجستيك والموارد
- المواد والمستلزمات
- تأكيد المورد
- حركة السيارات
- الكشوفات والأسماء معتمدة
- التنسيق مع الإعلام
- طباعة الاعلاميات
- مراحل التصوير — قبل وأثناء

### مجموعة 2: 📸 التحضيرات الميدانية
- تبليغ المستفيدين
- التنسيق مع الرقابة
- التنسيق مع المندوب / المخيم
- الموقع جاهز ومهيأ / الاعلام
- كود الكشوفات والكوبونات

### مجموعة 3: 👷 الموارد البشرية
(بنود has_person_field = true — تحمل اسم شخص توثيقياً، مثل "الاستقبال — جهاد أبو الريش")
- الاستقبال
- المطابقة
- التسليم
- المستودع
- التوثيق
- تقرير ختامي + رابط توثيق
- كشوفات توقيع ختامية

> ملاحظة: هذه نقطة بداية. الديناميكية تسمح لأدمن النظام بالتعديل الكامل (إضافة/حذف/إعادة ترتيب).

## واجهة إدارة قائمة التحقق (Admin Panel)

يملك هذا الجزء فقط مستخدم بصلاحية `checklist_admin.manage` (= أدمن النظام افتراضياً):
- إضافة مجموعة جديدة (اسم + ترتيب)
- تعديل اسم مجموعة، تفعيل/تعطيل
- إضافة بند لمجموعة (اسم + has_person_field + ترتيب)
- تعديل بند، تفعيل/تعطيل
- إعادة ترتيب (drag-and-drop أو أزرار أعلى/أسفل)

**تحذير مهم:** تعطيل بند أو مجموعة **لا يحذف** قيمها الموجودة في `project_checklist_values` — فقط يُخفيها من الفورم للمشاريع الجديدة. البيانات القديمة تبقى للتاريخ.
